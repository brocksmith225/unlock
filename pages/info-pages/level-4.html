Insecure Direct Object Reference

The Vulnerability

Insecure Direct Object Reference is another common flaw in modern websites. A direct object reference is a tool used by developers to request that a file be sent from the server to the end user. If this is exposed in a GET request and not properly validated, it can be exploited

How to Exploit It

Attackers can exploit Insecure Direct Object Reference by asking the server for a file they are allowed to access then modifying the GET request to ask for a file they do not have permission to. This was achieved in the previous level by requesting your account's 1040 form then modifying the GET request to ask for another user's 1040 form.

How to Prevent It

Insecure Direct Object Reference can be prevented by authenticating the user whenever they ask for a particular file. Only by authenticating the user with each and every request can a developer be entirely sure that no Insecure Direct Object References exist.
