SQL Injection

The Vulnerability

Injection Flaws are another common flaw in modern websites. It can show up anywhere that users can input unfiltered text. This particular flavor, SQL Injection, is the most commonly found Injection Flaw.

How to Exploit It

Injection Flaws are usually some of the easiest vulnerabilities to exploit. The attacker first must try to figure out what command their input is being put into, then inject code that will change the functionality of said command. For instance, SQL Injeciton usually involves a UNION command that allows the attacker to write their own query after the developer's query.

How to Prevent It

Injection Flaws are very simple to prevent. All you have to do as a developer is cleanse the input, making sure to escape any control characters (i.e. ' or ; in SQL).
