Broken Authentication and Session Management

The Vulnerability

Broken Authentication and Session Management is a common flaw in modern websites. Most commonly it occurs with compromised or insecure passwords; this is how it occurred in the previous level. It can also occur when session IDs are unprotected an can be accessed by attackers.

How to Exploit It

Broken Authentication and Session Management, when it involves insecure passwords, can easily be exploited using any password cracking attack such as brute force (attacks that try every possible combination of letters, numbers, and symbols) or dictionary attacks (attacks that tries passwords from a specially created set of common passwords). When it involves exposed session IDs, attackers can find the session ID of another user with some kind of man-in-the-middle attack and forge a POST request using the compromised session ID.

How to Prevent It

Luckily, Broken Authentication and Session Management is quite easy to prevent. Adding password complexity and lenth requirements helps to decrease the possibility of insecure passwords. Using a trusted, pre-existing framework for session management will help to secure session IDs.
