Broken Access Control

The Vulnerability

Broken Access Control is another common flaw in modern websites. It occurs when the developer does not properly authenticate the user before they complete an action. This allows anonymous users to complete actions only meant for logged in users or normal users to complete actions only meant for administrators.

How to Exploit It

Broken Access Control can be tough to spot, but it isn't necessarily tought to exploit. An attacker can simply try to complete an action they shouldn't have access to, like trying to transfer money as an anonymous user.

How to Prevent It

Broken Access Control can be very tough to prevent since one action that's not properly authenticated could be disastrous. In order to best ensure that this vulnerability doesn't occur, developers should extensively document their access control plan.
